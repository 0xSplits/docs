# Warehouse

Warehouse as the name suggests is the place where all the funds in the split
ecosystem pile up. It is built on top of
[`ERC6909`](https://eips.ethereum.org/EIPS/eip-6909) and
[`ERC6909X`](https://github.com/frangio/ERCs/blob/4d765d71940db25a901a592a17aa8e2a16b6efd6/ERCS/eip-xxxx.md).
Any user earning from the splits ecosystem of contracts that deposit to the
warehouse can access their assets directly from the warehouse.

- [Github](https://github.com/0xSplits/splits-contracts-monorepo/tree/main/packages/splits-v2)
- [Contracts & Natspec](https://github.com/0xSplits/splits-contracts-monorepo/blob/main/packages/splits-v2/src/SplitsWarehouse.sol)
- [Audit](https://github.com/0xSplits/splits-contracts-monorepo/blob/main/audits/splits-v2.md)

## How it works

- The warehouse has a unique id for each token. This id is the `uint256`
  representation of the token `address`.
- For each deposited token into the warehouse the contract returns a receipt
  which can be burnt to regain the original deposit. All deposits will always be
  1:1 with the receipt.
- Users can transfer these tokens just like ERC20 tokens by calling the
  warehouse `transfer` and `transferFrom` functions.
- Users who want to automate withdrawals from the warehouse can do so by setting
  a withdraw incentive. This will incentives bots to withdraw funds for the user
  directly into their account.
- Warehouse is an immutable contract that is not owned by the team.
- We recommend reviewing the [contracts flow of funds](/core#flow-of-funds).
